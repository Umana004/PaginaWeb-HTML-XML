Introducción

Este sitio web tiene como propósito mostrar los conceptos fundamentales de los lenguajes HTML (HyperText Markup Language) y XML (Extensible Markup Language), basándose en las especificaciones oficiales: HTML Living Standard y XML 1.1 (Second Edition).

A lo largo de este documento se explican sus estructuras básicas, similitudes, diferencias y ejemplos prácticos, reflejando el conocimiento adquirido sobre el uso correcto de estos lenguajes de marcado.

Estructura básica de un documento HTML

1. Infraestructura del documento

Según la especificación HTML Living Standard (sección 4.1.1), todo documento HTML inicia con la declaración <!DOCTYPE html>, que indica al navegador que el archivo cumple con el estándar HTML5. 

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Ejemplo HTML</title>
</head>
<body>
    <h1>Encabezado principal</h1>
    <p>Este es un párrafo de ejemplo.</p>
</body>
</html>

Este formato define claramente la estructura jerárquica del documento: un elemento raíz <html> que contiene los elementos <head> y <body>.

2. Etiquetas más populares

  • <h1>...</h1>: Encabezados de distintos niveles.
  • <p>...</p>: Párrafos de texto.
  • <a>...</a>: Enlaces a otras páginas.
  • <img>: Inserta imágenes.
  • <div> y <span>: Contenedores para agrupar contenido.

3. Características principales

  • Usa etiquetas predefinidas, no se pueden inventar nuevas.
  • Está orientado a la visualización en navegadores.
  • No distingue mayúsculas/minúsculas (aunque se recomienda minúsculas).
  • Los navegadores son tolerantes a errores de sintaxis.

Estructura básica de un documento XML

1. Definición de un documento bien formado

Según la especificación XML 1.1 (Second Edition), sección 2.8, un documento XML se considera bien formado cuando:

  • Tiene un único elemento raíz.
  • Las etiquetas están correctamente anidadas y cerradas.
  • Los atributos están entre comillas.
  • No contiene errores de sintaxis.

2. Infraestructura del documento

La estructura general de un archivo XML es la siguiente:

<?xml version="1.0" encoding="UTF-8"?>
<libro>
    <titulo>Programación Web</titulo>
    <autor>Steven Umaña</autor>
    <año>2025</año>
</libro>

XML permite definir etiquetas personalizadas según el tipo de datos. Por ejemplo, en el ejemplo anterior, <libro> es el elemento raíz y dentro se definen los subelementos <titulo>, <autor> y <año>.

3. Características principales

  • Las etiquetas son personalizables.
  • Es sensible a mayúsculas y minúsculas.
  • Está diseñado para almacenar y transportar información, no para mostrarla.
  • Debe cumplir estrictamente las reglas de sintaxis.

Diferencias y similitudes entre HTML y XML

Aspecto HTML XML
Propósito principal Mostrar información en navegadores web. Almacenar y transportar información estructurada.
Etiquetas Predefinidas (ej. <p>, <img>). Definidas por el usuario (ej. <persona>, <nombre>).
Sensibilidad a mayúsculas No sensible. Sensible (ej. <Nombre> ≠ <nombre>).
Cierre de etiquetas Algunas no requieren cierre (ej. <br>). Todas deben cerrarse correctamente.
Tolerancia a errores El navegador intenta corregirlos. Un error invalida todo el documento.
Uso principal Diseño y presentación web. Intercambio de datos entre sistemas.

Similitudes

  • Ambos usan etiquetas y estructura jerárquica.
  • Ambos pueden incluir atributos en las etiquetas.
  • Ambos requieren una sintaxis bien formada.
  • Ambos son lenguajes de marcado, no de programación.
